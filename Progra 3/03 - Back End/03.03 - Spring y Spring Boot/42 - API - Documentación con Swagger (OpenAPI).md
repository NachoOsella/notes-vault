# API - Documentacion con Swagger (OpenAPI)

## Definicion
Swagger/OpenAPI es un estandar y conjunto de herramientas para describir y visualizar APIs REST de forma clara.

## Explicacion
- *Que problema resuelve*
    Evita documentacion manual desactualizada y mejora consumo de la API.
- *Como funciona por arriba*
    Se define especificacion OpenAPI (JSON/YAML) y Swagger UI muestra endpoints, parametros y respuestas.
- *Que implica / que permite*
    Exploracion interactiva, contratos mas claros y generacion de clientes.

## Palabras clave
- OpenAPI
- Swagger UI
- especificacion
- contrato API

## Comparaciones tipicas
- vs README manual: OpenAPI es estructurado y mas facil de mantener.
- vs [[16 - Spring MVC - @RestController]]: `@RestController` implementa endpoints; OpenAPI los documenta.

## Errores comunes
- Publicar documentacion sin reflejar codigos de error reales.
- No versionar la especificacion de la API.
