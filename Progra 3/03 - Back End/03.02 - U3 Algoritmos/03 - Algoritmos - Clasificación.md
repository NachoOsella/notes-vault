# Algoritmos - Clasificación

## Definición
Existen diferentes tipos de algoritmos y se clasifican en categorías según el tipo de problema que resuelven.
### Ejemplos
- *Algoritmos de búsqueda*: lineal, binaria, hashing
- *Algoritmos de ordenamiento*: burbuja, selección, inserción, mergesort, quicksort
- *Algoritmos de grafos*: Dijkstra, Floyd-Warshall, Prim, Kruskal
- *Algoritmos de programación dinámica*: Fibonacci, mochila, cadena de matrices
- *Algoritmos de backtracking*: N-reinas, Sudoku
- *Algoritmos de divide y vencerás*: mergesort, quicksort, Strassen

## Explicación

- *Qué problema resuelve*
    Organiza y estructura el conocimiento sobre algoritmos permitiendo agruparlos según diferentes criterios para elegir el más adecuado según el contexto
- *Cómo funciona por arriba*
    Establece taxonomías basadas en múltiples dimensiones: tipo de problema, estrategia de solución, eficiencia, estructura de datos utilizada, etc.
- *Qué implica / qué permite*
    Facilita la comunicación entre programadores, permite estimar recursos necesarios, ayuda a seleccionar algoritmos apropiados y comprender límites teóricos

## Preguntas de examen
- ¿Qué ventaja práctica tiene clasificar algoritmos antes de implementar?
- ¿Qué criterio usarías primero para elegir un algoritmo en un problema real?

## Errores comunes
- Mezclar criterios distintos como si fueran equivalentes.
- Elegir algoritmo por costumbre sin analizar problema ni escala.

## Mini-ejemplo (mental)
- Elegir ruta en GPS: primero definís el tipo de problema (ruta), luego la estrategia (rápida/segura) y finalmente el costo esperado.
