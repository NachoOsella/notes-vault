# Tailwind - Sintaxis básica

## Definición
La sintaxis de Tailwind es muy amplia, algunos de los ejemplos más comunes incluyen:
- Clases para márgenes y paddings: `m-4`, `p-2`
- Clases para colores: `bg-blue-500`, `text-white`
- Clases para tipografía: `text-lg`, `font-bold`
- Clases para layout: `flex`, `grid`, `items-center`
- Clases para responsividad: `md:text-center`, `lg:p-8`

## Explicación
- *Qué problema resuelve*
    Facilita la aplicación rápida de estilos sin necesidad de escribir CSS personalizado, permitiendo un desarrollo más ágil y consistente.
- *Cómo funciona por arriba*
    Utiliza una convención de nombres clara y predecible para las clases, donde cada clase representa una propiedad CSS específica con un valor determinado.
- *Qué implica / qué permite*
    Permite a los desarrolladores aplicar estilos directamente en el HTML, promoviendo un enfoque de diseño basado en utilidades y facilitando la creación de interfaces responsivas y personalizadas.

## Palabras clave
- Clases utilitarias
- Convención de nombres
- Aplicación directa en HTML
- Desarrollo ágil

## Comparaciones típicas
- vs [[CSS - Propiedades comunes]]: Tailwind expresa propiedades como clases; CSS las expresa como reglas.
- vs [[Tailwind - Conceptos clave]]: conceptos es el modelo mental; sintaxis es la aplicación concreta en el HTML.

## Errores comunes
- Confundir las clases de Tailwind con clases personalizadas.
- Pensar que todas las propiedades CSS están disponibles como clases Tailwind.
- No entender la convención de nombres y cómo se estructuran las clases.
