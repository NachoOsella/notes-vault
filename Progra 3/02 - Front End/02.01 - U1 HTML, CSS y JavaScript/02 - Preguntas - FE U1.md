## Respuestas (opcional)

### ¿Qué es HTML?
Respuesta: Lenguaje de marcado para estructurar contenido web con etiquetas. Ver: [[03 - HTML - Definición]]

### ¿Para qué sirve HTML?
Respuesta: Sirve para organizar el contenido de una página web (títulos, párrafos, enlaces, imágenes y formularios). Ver: [[03 - HTML - Definición]]

### ¿Cuál es la diferencia entre HTML y CSS?
Respuesta: HTML define la estructura y el contenido; CSS define la presentación visual. Ver: [[03 - HTML - Definición]] y [[06 - CSS - Definición]]

### ¿Cuál es la diferencia entre etiquetas semánticas y no semánticas?
Respuesta: Semánticas aportan significado (mejoran accesibilidad/SEO); no semánticas son genéricas (`div`, `span`). Ver: [[05 - HTML - Etiquetas semánticas]]

### ¿Cuáles son las etiquetas HTML más importantes?
Respuesta: Las más usadas son `html`, `head`, `body`, `h1`-`h6`, `p`, `a`, `img`, `ul`/`ol`/`li`, `div`, `span`, `form` e `input`.

### ¿Qué es CSS y qué hace?
Respuesta: CSS es el lenguaje de estilos que define colores, tipografía, espaciado y disposición visual de los elementos HTML.

### ¿Qué es un selector en CSS?
Respuesta: Un patrón que apunta a elementos HTML para aplicarles estilos (tipo, clase, id, atributo, etc.). Ver: [[07 - CSS - Selectores]]

### ¿Qué diferencia hay entre selectores y propiedades?
Respuesta: El selector indica a qué elemento se aplica el estilo; la propiedad indica qué aspecto se modifica, como `color` o `margin`.

### ¿Qué es el box model?
Respuesta: Modelo de tamaño/espacio: content + padding + border + margin. Ver: [[09 - CSS - Box Model]]

### ¿Qué es diseño responsivo?
Respuesta: Técnica para que la UI se adapte a distintos tamaños de pantalla usando media queries y layouts flexibles. Ver: [[10 - CSS - Diseño responsivo]]

### ¿Cuáles son las propiedades CSS más comunes?
Respuesta: Entre las más comunes están `color`, `background-color`, `font-size`, `margin`, `padding`, `border`, `width`, `height`, `display` y `position`.

### ¿Qué es una función?
Respuesta: Bloque reutilizable que recibe parámetros y puede retornar un valor. Ver: [[14 - JS - Funciones]]

### ¿Cuál es la diferencia entre parámetros y valor de retorno?
Respuesta: Los parámetros son datos de entrada de la función; el valor de retorno es el resultado que devuelve al terminar.

### ¿Para qué sirven las estructuras de control?
Respuesta: Controlan el flujo del programa con decisiones (if/switch) y repeticiones (for/while). Ver: [[13 - JS - Estructuras de control]]

### ¿Cuándo usar `if/else` vs un loop?
Respuesta: Usá `if/else` para decidir entre caminos según condiciones; usá loops para repetir acciones o recorrer colecciones de datos.

### ¿Qué es Tailwind?
Respuesta: Framework CSS utility-first: componés estilos con clases en el HTML en lugar de escribir muchas reglas CSS. Ver: [[11 - Tailwind - Conceptos clave]]

### ¿Qué significa “utility-first”?
Respuesta: Significa construir estilos con clases utilitarias pequeñas y combinables directamente en el HTML.

### ¿Qué diferencia hay entre CSS puro y Tailwind?
Respuesta: En CSS puro escribís reglas propias en archivos `.css`; en Tailwind aplicás utilidades predefinidas mediante clases.

### ¿Cómo se aplica un estilo en Tailwind?
Respuesta: Con clases en el atributo `class`, por ejemplo `class="bg-blue-500 p-4 rounded"`. Ver: [[12 - Tailwind - Sintaxis básica]]
