# HTML - Etiquetas semánticas

## Definición
Las etiquetas semánticas en HTML son elementos que describen el significado del bloque de contenido. Facilitan la comprensión del desarrollador y mejoran la accesibilidad y el SEO. Ejemplos incluyen `<header>`, `<nav>`, `<article>`, `<section>`, `<footer>`, entre otras.

## Explicación
- *Qué problema resuelve*
    Proporcionar significado y estructura al contenido web, mejorando la accesibilidad y la interpretación por parte de motores de búsqueda.
- *Cómo funciona por arriba*
    Las etiquetas semánticas definen claramente la función de diferentes partes de una página web. Por ejemplo, `<nav>` indica una sección de navegación, mientras que `<article>` representa un contenido independiente y autocontenido.
- *Qué implica / qué permite*
    Permite a los desarrolladores crear páginas web más organizadas y accesibles, facilitando la navegación y comprensión del contenido tanto para usuarios como para motores de búsqueda.

## Comparaciones típicas
- vs [[04 - HTML - Principales etiquetas]]: Las etiquetas semánticas aportan significado al contenido, mientras que las etiquetas generales como `<div>` y `<span>` son más flexibles pero carecen de significado específico.
- vs [[03 - HTML - Definición]]: HTML define la estructura; las etiquetas semánticas son una forma de expresar esa estructura con significado.

## Errores comunes
- Usar etiquetas no semánticas como `<div>` en lugar de etiquetas semánticas adecuadas.
- No aprovechar las ventajas de accesibilidad y SEO que ofrecen las etiquetas semánticas.
