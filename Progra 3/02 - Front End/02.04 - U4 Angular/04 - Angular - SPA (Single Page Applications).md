# Angular - SPA (Single Page Applications)

## Definición

Una SPA es una aplicación web que carga una única página base y actualiza dinámicamente su contenido sin recargar todo el documento en cada interacción.

## Explicación

- *Qué problema resuelve*
    Reduce recargas completas de página, mejorando fluidez, tiempos de respuesta percibidos y experiencia de usuario.

- *Cómo funciona por arriba*
    El frontend maneja navegación y renderizado parcial; el backend suele exponer APIs para datos.

- *Qué implica / qué permite*
    Facilita interfaces más interactivas, arquitectura modular y evolución incremental de funcionalidades.

## Palabras clave

- Single Page Application
- Navegación fluida
- Renderizado dinámico
- API
- UX

## Comparaciones típicas

- vs MPA (multi-page): en MPA cada navegación recarga documento completo; en SPA no.
- vs [[03 - Angular - Introducción]]: Angular es una opción tecnológica para construir SPA.

## Errores comunes

- Creer que “SPA” implica que no existe backend.
- Suponer que toda interacción en SPA es instantánea sin buena gestión de datos.
